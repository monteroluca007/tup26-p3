# Switch Expression y Pattern Matching en C#

El **pattern matching** (coincidencia de patrones) es la capacidad de examinar un valor y ramificar la ejecución según su **estructura**, **tipo** o **contenido** —no sólo su valor exacto. En C# moderno es una de las características más potentes para escribir código expresivo y sin ramas anidadas.

El **switch expression** es la forma principal de usar pattern matching: produce un valor directamente, en lugar de ejecutar sentencias.

```csharp
// Antes — imperativo, muchas variables temporales
string resultado;
if (figura is Circulo c) {
    resultado = $"Círculo de radio {c.Radio}";
} else if (figura is Rectangulo r) {  
    resultado = $"Rectángulo {r.Ancho}×{r.Alto}";
} else {
    resultado = "Figura desconocida";
}

// Ahora — switch expression con patterns
string resultado = figura switch {
    Circulo    c => $"Círculo de radio {c.Radio}",
    Rectangulo r => $"Rectángulo {r.Ancho}×{r.Alto}",
    _            => "Figura desconocida"
};
```

---

## 1. Anatomía del switch expression

```csharp
//  ┌─ expresión a examinar
//  │
valor switch {
    patron1             => resultado1, // brazo
    patron2 when guarda => resultado2, // brazo con guardia
    _                   => porDefecto  // wildcard — siempre al final
//              │               │
//              └── guardia     └── resultado del brazo
};
```

Diferencias fundamentales con la instrucción `switch` clásica:

| Aspecto        | `switch` instrucción   | `switch` expression         |
|----------------|------------------------|-----------------------------|
| Produce        | Ejecuta sentencias     | Un valor                    |
| `break`        | Obligatorio            | No existe                   |
| `default`      | Keyword                | `_` (wildcard)              |
| Exhaustividad  | No verificada          | Verificada en compilación   |
| Asignación     | Con variable temporal  | Directamente                |

## 2. Constant pattern — valores literales

El patrón más simple: coincide si el valor es igual a una constante.

```csharp
int codigo = 404;

string mensaje = codigo switch {
    200 => "OK",
    201 => "Creado",
    204 => "Sin contenido",
    301 => "Movido permanentemente",
    400 => "Solicitud incorrecta",
    401 => "No autorizado",
    403 => "Prohibido",
    404 => "No encontrado",
    500 => "Error interno del servidor",
    _   => $"Código desconocido: {codigo}"
};
```

También funciona con strings, chars, bools y enums:

```csharp
string comando = "salir";

string respuesta = comando switch {
    "ayuda"   or "help" or "?" => MostrarAyuda(),
    "salir"   or "exit"        => "Hasta luego.",
    "limpiar" or "cls"         => Limpiar(),
    ""                         => "Comando vacío.",
    _                          => $"Comando desconocido: '{comando}'"
};

// Con char
char operador = '+';
Func<double, double, double> op = operador switch {
    '+' => (a, b) => a + b,
    '-' => (a, b) => a - b,
    '*' => (a, b) => a * b,
    '/' => (a, b) => b == 0 ? throw new DivideByZeroException() : a / b,
    _   => throw new ArgumentException($"Operador desconocido: {operador}")
};
```

## 3. Type pattern — coincidencia por tipo

Verifica si el valor **es** de un tipo específico y opcionalmente lo **declara** en una variable:

```csharp
object valor = 3.14;

// Sin variable — sólo verifica el tipo
string descripcion1 = valor switch {
    int    => "entero",
    double => "decimal de doble precisión",
    string => "cadena de texto",
    bool   => "booleano",
    null   => "nulo",
    _      => "tipo desconocido"
};

// Con variable — verifica y declara en un paso
string descripcion2 = valor switch {
    int    n => $"entero: {n}",
    double d => $"decimal: {d:F2}",
    string s => $"cadena de {s.Length} caracteres",
    bool   b => $"booleano: {(b ? "verdadero" : "falso")}",
    null     => "nulo",
    _        => $"tipo: {valor.GetType().Name}"
};
```

### 3.1 Polimorfismo con type pattern

```csharp
abstract record Figura(string Color);
record Circulo(string Color, double Radio) : Figura(Color);
record Rectangulo(string Color, double Ancho, double Alto) : Figura(Color);
record Triangulo(string Color, double Base, double Altura) : Figura(Color);
record Punto(string Color) : Figura(Color);

static double Area(Figura f) => f switch {
        Circulo    c => Math.PI * c.Radio * c.Radio,
        Rectangulo r => r.Ancho * r.Alto,
        Triangulo  t => t.Base * t.Altura / 2,
        Punto        => 0,
        _            => throw new ArgumentException($"Figura no soportada: {f.GetType().Name}")
    };

static string Describir(Figura f) => f switch {
        Circulo    { Radio: 0 }          => "Punto degenerado",
        Circulo    c                     => $"Círculo r={c.Radio:F1}",
        Rectangulo { Ancho: var a, Alto: var h } when a == h
                                         => $"Cuadrado de lado {a}",
        Rectangulo r                     => $"Rectángulo {r.Ancho}×{r.Alto}",
        Triangulo  t                     => $"Triángulo b={t.Base} h={t.Altura}",
        _                                => "Figura simple"
    };
```

> El orden importa: los patrones más específicos deben ir **antes** que los más generales. Si `Circulo { Radio: 0 }` estuviera después de `Circulo c`, nunca coincidiría.

## 4. Property pattern — coincidencia por propiedades

Examina las propiedades del objeto directamente en el patrón:

```csharp
record Pedido(int Id, decimal Total, string Estado, bool EsUrgente, string Pais);

// Patrón de propiedad simple
static string PrioridadPedido(Pedido p) => p switch {
        { Estado: "cancelado" }              => "Ignorar",
        { EsUrgente: true, Total: > 50_000 } => "Prioridad máxima",
        { EsUrgente: true }                  => "Alta",
        { Total: > 100_000 }                 => "Alta por monto",
        { Pais: "AR", Total: > 10_000 }      => "Media",
        _                                    => "Normal"
    };

// Propiedades anidadas — se accede con notación de punto dentro del patrón
record Direccion(string Ciudad, string Pais, string CodigoPostal);
record Cliente(string Nombre, Direccion? Direccion, bool EsVip);

static decimal Descuento(Cliente c) => c switch {
        { EsVip: true, Direccion.Pais: "AR" } => 0.20m,
        { EsVip: true }                       => 0.15m,
        { Direccion.Ciudad: "Tucumán" }       => 0.05m,
        { Direccion: null }                   =>    0m,
        _                                     =>    0m
    };
```

### 4.1 Capturar el valor de una propiedad — `var`

```csharp
record Producto(string Nombre, decimal Precio, int Stock, string Categoria);

static string ClasificarProducto(Producto p) => p switch {
        // Capturar el valor de la propiedad en una variable local
        { Stock: 0 }                                 => $"{p.Nombre}: Sin stock",
        { Precio: var precio } when precio > 100_000 => $"{p.Nombre}: Precio VIP ${precio:N0}",
        { Nombre: var n, Categoria: "electrónica" }  => $"{n} (electrónica)",
        { Stock: var s } when s < 5                  => $"{p.Nombre}: Stock crítico ({s})",
        _                                            => p.Nombre
    };
```

### 4.2 Property pattern con `is`

El property pattern también funciona fuera del switch con `is`:

```csharp
Cliente cliente = new("Ana", new("Tucumán", "AR", "4000"), true);

if (cliente is { EsVip: true, Direccion.Pais: "AR" })
    Console.WriteLine("Cliente VIP argentino — descuento especial");

// Declarar variables mientras se verifica
if (cliente is { Nombre: var nombre, EsVip: true })
    Console.WriteLine($"Bienvenida, {nombre}");
```

## 5. Relational pattern — comparaciones

Permite usar `<`, `>`, `<=`, `>=` directamente como patrón:

```csharp
static string ClasificarEdad(int edad) => edad switch {
        <  0 => throw new ArgumentOutOfRangeException(),
        <  3 => "Bebé",
        < 13 => "Niño",
        < 18 => "Adolescente",
        < 30 => "Joven adulto",
        < 60 => "Adulto",
        < 80 => "Adulto mayor",
        _    => "Anciano"
    };

static string ClasificarTemperatura(double temp) => temp switch {
        < -10 => "Helado",
        <   0 => "Bajo cero",
        <  15 => "Frío",
        <  25 => "Templado",
        <  35 => "Caluroso",
        _     => "Muy caluroso"
    };

// Con decimal
static string BancoInteresAnual(decimal monto) => monto switch {
        <=        0 => throw new ArgumentOutOfRangeException(),
        <   100_000 => "2.5%",
        <   500_000 => "3.2%",
        < 1_000_000 => "4.0%",
        _           => "4.8% — requiere asesor"
    };
```

## 6. Logical pattern — `and`, `or`, `not`

Combina patrones con operadores lógicos:

```csharp
// and — ambos patrones deben coincidir
static bool EsAdultoJoven(int edad) => edad is >= 18 and <= 35;

// or — al menos uno debe coincidir
static bool EsFinDeSemana(DayOfWeek dia)
    => dia is DayOfWeek.Saturday or DayOfWeek.Sunday;

// not — el patrón NO debe coincidir
static bool EsLaboral(DayOfWeek dia)
    => dia is not (DayOfWeek.Saturday or DayOfWeek.Sunday);

// Combinaciones en switch expression
static string ZonaHoraria(int hora) => hora switch {
        >=  0 and <   6 => "Madrugada",
        >=  6 and <  12 => "Mañana",
        >= 12 and <  14 => "Mediodía",
        >= 14 and <  18 => "Tarde",
        >= 18 and <  21 => "Noche",
        >= 21 and <= 23 => "Noche tardía",
        _               => throw new ArgumentOutOfRangeException()
    };

// not con type pattern
static void Procesar(object obj) {
    if (obj is not string)
        Console.WriteLine("No es un string");

    if (obj is not null and not string)
        Console.WriteLine("No es null ni string");
}
```

## 7. Tuple pattern — múltiples valores simultáneos

Permite hacer coincidir sobre **varios valores a la vez**:

```csharp
// Dos variables a la vez
static string Describir(bool esDia, bool llueve) => (esDia, llueve) switch {
        (true,  false) => "Día soleado",
        (true,  true ) => "Día lluvioso",
        (false, false) => "Noche estrellada",
        (false, true ) => "Noche tormentosa"
    };

// Tres variables
static string ModoTransporte(bool tieneAuto, bool hayColectivo, double distanciaKm)
    => (tieneAuto, hayColectivo, distanciaKm) switch {
        (_    , _   , < 2) => "Caminá",
        (true , _   , _  ) => "Manejá",
        (false, true, _  ) => "Colectivo",
        _                  => "Remís"
    };

// Con enums — tabla de verdad semántica
static decimal Precio(Categoria cat, Temporada temp) => (cat, temp) switch {
        (Categoria.Ropa,        Temporada.Alta) => 1.3m,
        (Categoria.Ropa,        Temporada.Baja) => 0.7m,
        (Categoria.Electronica, Temporada.Alta) => 1.1m,
        (Categoria.Electronica, Temporada.Baja) => 0.9m,
        (Categoria.Alimentos,   _             ) => 1.0m,   
        _                                       => 1.0m // _ ignora Temporada
    };

// Tuple pattern con relational
static string Cuadrante(double x, double y) => (x, y) switch {
        (> 0, > 0) => "I",
        (< 0, > 0) => "II",
        (< 0, < 0) => "III",
        (> 0, < 0) => "IV",
        (  0,   0) => "Origen",
        (  0,   _) => "Eje Y",
        (  _,   0) => "Eje X",
        _          => "Indefinido"   // nunca se alcanza, por exhaustividad
    };
```

## 8. Positional pattern — deconstrucción

Si el tipo tiene un método `Deconstruct`, el patrón puede desestructurarlo directamente:

```csharp
// record genera Deconstruct automáticamente
record Punto(double X, double Y);
record Rango(int Min, int Max);

static string PosicionPunto(Punto p) => p switch {
        (    0,     0) => "Origen",
        (var x,     0) => $"Sobre el eje X en {x}",
        (    0, var y) => $"Sobre el eje Y en {y}",
        (>   0, >   0) => "Cuadrante I",
        (<   0, >   0) => "Cuadrante II",
        (<   0, <   0) => "Cuadrante III",
        (>   0, <   0) => "Cuadrante IV",
        _              => "Indefinido"
    };

static bool ContieneValor(Rango r, int valor)
    => r is (var min, var max) && valor >= min && valor <= max;

// Deconstruct personalizado en una clase
public class Temperatura(double celsius) {
    public double Celsius    => celsius;
    public double Fahrenheit => celsius * 9 / 5 + 32;

    // Método Deconstruct — habilita el positional pattern
    public void Deconstruct(out double c, out double f)
        => (c, f) = (Celsius, Fahrenheit);
}

static string ClasificarTemp(Temperatura t) => t switch {
        (<   0,     _) => "Bajo cero",
        (_    , > 100) => "Más de 100°F",
        (var c, var f) => $"{c}°C / {f}°F"
    };
```

## 9. List pattern — coincidencia sobre colecciones (C# 11+)

Permite hacer coincidir sobre arrays y listas por su **estructura y contenido**:

```csharp
// Sintaxis: [patron1, patron2, ..., patronN]
// ..  = "cualquier cantidad de elementos" (slice pattern)

static string AnalizarLista(int[] nums) => nums switch {
        []                => "vacía",
        [0]               => "contiene sólo un cero",
        [var x]           => $"un elemento: {x}",
        [var x, var y]    => $"dos elementos: {x} y {y}",
        [1, 2, 3]         => "exactamente [1, 2, 3]",
        [> 0, > 0, > 0]   => "tres positivos",
        [var primero, ..] => $"empieza con {primero}",
        _                 => "otra cosa"
    };

// .. captura el resto (slice pattern)
static string ExaminarPrimeroYUltimo(int[] nums) => nums switch {
        []                              => "vacía",
        [var unico]                     => $"único: {unico}",
        [var primero, .., var ultimo]   => $"del {primero} al {ultimo}",
    };

// Combinar con otros patrones
static bool EsSecuenciaAscendente(int[] nums) => nums switch {
        [] or [_]                    => true,
        [var a, var b, .. var resto] => a < b && EsSecuenciaAscendente([b, .. resto]),
        _                            => false
    };

// Con strings
static string ClasificarArgs(string[] args) => args switch {
        []           => "Sin argumentos",
        ["--help"] or ["-h"]    => "Ayuda",
        ["--version"]           => "Versión",
        [var cmd]               => $"Comando: {cmd}",
        [var cmd, var arg]      => $"Comando: {cmd}, Arg: {arg}",
        [var cmd, .. var resto] => $"Comando: {cmd} con {resto.Length} argumentos extra",
    };

// Validar estructuras específicas
static bool EsComandoValido(string[] tokens) => tokens switch {
        ["get",  _, ..] => true,    // get <recurso> [opciones]
        ["post", _, _]  => true,    // post <recurso> <cuerpo>
        ["delete", _]   => true,    // delete <id>
        _               => false
    };
```

## 10. `var` pattern — captura siempre

El patrón `var` siempre coincide y captura el valor en una nueva variable. Es útil dentro de patrones compuestos para capturar una parte del valor:

```csharp
// var siempre coincide — equivale a un wildcard que captura
static void ProcesarValor(object? obj) {
    if (obj is var valor)   // siempre true — captura obj en valor
        Console.WriteLine(valor?.GetType().Name ?? "null");
}

// Más útil dentro de patrones complejos para capturar y usar en la guardia
static string ClasificarString(string? s) => s switch {
        null                           => "nulo",
        var t when t.Length == 0       => "vacío",
        var t when t.Length < 5        => $"corto: '{t}'",
        var t when t.All(char.IsDigit) => $"numérico: {t}",
        var t                          => $"texto: '{t[..5]}...'"
};

// Capturar una propiedad calculada para usarla en la guardia
record Empleado(string Nombre, decimal Salario, int Años);

static string CategoriaEmpleado(Empleado e) => e switch {
        { Años: var a, Salario: var s } when a > 10 && s > 150_000 => "Ejecutivo senior",
        { Años: var a } when a > 10                                => "Senior",
        { Salario: var s } when s > 100_000                        => "Alto valor",
        { Nombre: var n } when n.StartsWith("Dr.")                 => "Especialista",
        _                                                          => "Empleado estándar"
    };
```

## 11. Guardias `when` — condición adicional

La guardia `when` agrega una condición booleana **después** de que el patrón coincide. Si la guardia es falsa, el brazo se descarta y se evalúa el siguiente:

```csharp
record Transaccion(decimal Monto, string Moneda, string TipoCliente, DateTime Fecha);

static string ClasificarTransaccion(Transaccion t) => t switch {
        // El tipo y propiedades coinciden primero, luego se evalúa la guardia
        { Moneda: "USD", Monto: var m } when m > 10_000
            => "Transferencia USD de alto valor — requiere compliance",

        { TipoCliente: "VIP" } when t.Fecha.DayOfWeek is DayOfWeek.Saturday or DayOfWeek.Sunday
            => "Operación VIP en fin de semana — alertar gerencia",

        { Monto: var m, Moneda: "ARS" } when m > 1_000_000
            => "Gran operación en pesos",

        // when puede llamar a cualquier función
        _ when EsHorarioBancario(t.Fecha)
            => "Transacción en horario bancario",

        _
            => "Transacción estándar"
    };

static bool EsHorarioBancario(DateTime dt) => 
       dt.Hour >= 9 && dt.Hour < 18           &&
       dt.DayOfWeek is not DayOfWeek.Saturday &&
       dt.DayOfWeek is not DayOfWeek.Sunday;
```

> La guardia se evalúa **sólo si el patrón coincide**. Si el patrón no coincide, la guardia ni se ejecuta. Esto hace el orden importante: dos brazos con el mismo patrón pero distintas guardias son válidos.

## 12. Exhaustividad — cobertura de casos

El compilador verifica que el switch expression cubra todos los casos posibles. Si no lo hace, advierte o falla en compilación:

```csharp
// El compilador detecta que faltan casos
static string Describir(bool b) => b switch {
        true => "verdadero"
        // Error CS8509: switch expression does not handle all possible values
    };

// Correcto — todos los casos cubiertos
static string Describir(bool b) => b switch {
        true  => "verdadero",
        false => "falso"   // no necesita _ porque bool sólo tiene dos valores
    };

// Para enums — el compilador no sabe cuáles valores pueden llegar en runtime
// Siempre agregar _ o throw para valores inesperados
static string NombreMes(int mes) => mes switch {
        1  => "Enero",   2  => "Febrero",  3  => "Marzo",
        4  => "Abril",   5  => "Mayo",     6  => "Junio",
        7  => "Julio",   8  => "Agosto",   9  => "Septiembre",
        10 => "Octubre", 11 => "Noviembre",12 => "Diciembre",
        _  => throw new ArgumentOutOfRangeException(nameof(mes), $"Mes inválido: {mes}")
    };
```

## 13. Patrones con `is` fuera del switch

Todos los patrones funcionan también con el operador `is` en `if`, `while` y otras expresiones booleanas:

```csharp
object obj = new List<int> { 1, 2, 3 };

// Type pattern con is
if (obj is List<int> lista && lista.Count > 0)
    Console.WriteLine($"Lista con {lista.Count} elementos");

// Property pattern con is
record Persona(string Nombre, int Edad);
var persona = new Persona("Ana", 25);

if (persona is { Nombre: "Ana", Edad: >= 18 })
    Console.WriteLine("Ana es adulta");

// Negación con is not
if (persona is not { Edad: < 18 })
    Console.WriteLine("No es menor de edad");

// Combinar múltiples is en una condición
static bool EsApto(object? candidato) =>
    candidato is Persona { Edad: >= 18 and <= 65 }
             and not { Nombre: "" }
             and not null;

// is en while — procesar una secuencia heterogénea
object?[] items = [42, "texto", null, 3.14, true, "otro"];
foreach (var item in items) {
    if (item is int n and > 0)
        Console.WriteLine($"Entero positivo: {n}");
    else if (item is string { Length: > 0 } s)
        Console.WriteLine($"String no vacío: {s}");
    else if (item is null)
        Console.WriteLine("Nulo");
}
```

## 14. Combinando patrones — casos reales

### 14.1 Procesador de comandos CLI

```csharp
record Comando(string Nombre, string[] Args);

static string EjecutarComando(Comando cmd) => cmd switch {
        { Nombre: "help" or "h", Args: [] }
            => MostrarAyuda(),

        { Nombre: "get", Args: [var recurso] }
            => ObtenerRecurso(recurso),

        { Nombre: "get", Args: [var recurso, "--format", var fmt] }
            => ObtenerRecursoFormateado(recurso, fmt),

        { Nombre: "delete", Args: [var id] } when int.TryParse(id, out _)
            => EliminarRecurso(id),

        { Nombre: "list", Args: [] }
            => ListarTodos(),

        { Nombre: "list", Args: ["--filter", var filtro, ..] }
            => ListarFiltrado(filtro),

        { Nombre: var cmd, Args: var args } when args.Length > 10
            => $"Demasiados argumentos para '{cmd}'",

        { Nombre: var desconocido }
            => $"Comando desconocido: '{desconocido}'. Use 'help'."
    };
```

### 14.2 Calculadora de descuentos

```csharp
record Compra(
    decimal  Subtotal,
    int      CantidadItems,
    string   CategoriaCliente,   // "nuevo", "regular", "vip", "corporativo"
    string   Cupon,
    DateTime Fecha);

static decimal CalcularDescuento(Compra c) => c switch {
        // Cupones especiales primero — mayor prioridad
        { Cupon: "BLACKFRIDAY" }
            => c.Subtotal * 0.40m,

        { Cupon: "NUEVO2026", CategoriaCliente: "nuevo" }
            => c.Subtotal * 0.25m,

        // Corporativo con monto alto
        { CategoriaCliente: "corporativo", Subtotal: > 500_000 }
            => c.Subtotal * 0.20m,

        { CategoriaCliente: "corporativo" }
            => c.Subtotal * 0.15m,

        // VIP con condición de temporada
        { CategoriaCliente: "vip" } when c.Fecha.Month is 12 or 1
            => c.Subtotal * 0.15m,

        { CategoriaCliente: "vip", Subtotal: > 100_000 }
            => c.Subtotal * 0.12m,

        { CategoriaCliente: "vip" }
            => c.Subtotal * 0.10m,

        // Regular por cantidad de items
        { CategoriaCliente: "regular", CantidadItems: > 20 }
            => c.Subtotal * 0.08m,

        { CategoriaCliente: "regular", Subtotal: > 50_000 }
            => c.Subtotal * 0.05m,

        // Sin descuento
        _   => 0m
    };
```

### 14.3 Sistema de tipos de resultado

```csharp
// Unión discriminada con records y herencia
abstract record Resultado<T>;
record Exito<T>(T Valor)                        : Resultado<T>;
record Error<T>(string Mensaje, int Codigo = 0) : Resultado<T>;
record Pendiente<T>()                           : Resultado<T>;

static void ManejarResultado<T>(Resultado<T> r) where T : notnull {
    string descripcion = r switch {
        Exito<T> { Valor: var v }                => $"✓ {v}",
        Error<T> { Codigo: 404, Mensaje: var m } => $"✗ No encontrado: {m}",
        Error<T> { Codigo: 401 }                 => "✗ Sin autorización",
        Error<T> { Codigo: var c, Mensaje: var m } when c >= 500
                                                 => $"✗ Error del servidor ({c}): {m}",
        Error<T>   { Mensaje: var m }            => $"✗ Error: {m}",
        Pendiente<T>                             => "⏳ Pendiente...",
        _                                        => "? Estado desconocido"
    };

    Console.WriteLine(descripcion);
}

// Uso
ManejarResultado(new Exito<string>("Datos recibidos"));
ManejarResultado(new Error<string>("Usuario no existe", 404));
ManejarResultado(new Pendiente<string>());
```

## 15. Tabla de referencia de patrones

| Patrón       | Sintaxis                         | Coincide cuando                             |
| ------------ | -------------------------------- | ------------------------------------------- |
| Constante    | `42`, `"texto"`, `true`, `null`  | El valor es igual a la constante            |
| Tipo         | `string s`, `int n`, `Circulo c` | El valor es del tipo especificado           |
| Tipo sin var | `string`, `int`                  | El valor es del tipo (sin capturar)         |
| Propiedad    | `{ Nombre: "Ana", Edad: > 18 }`  | Las propiedades coinciden con los patrones  |
| Relacional   | `> 5`, `<= 10`, `>= 0`           | La comparación es verdadera                 |
| Lógico and   | `> 0 and < 100`                  | Ambos patrones coinciden                    |
| Lógico or    | `"a" or "b" or "c"`              | Al menos uno coincide                       |
| Lógico not   | `not null`, `not string`         | El patrón NO coincide                       |
| Tuple        | `(> 0, < 0)`, `(true, false)`    | Cada elemento coincide con su patrón        |
| Posicional   | `(var x, var y)`                 | Deconstrucción exitosa con patrones dados   |
| Lista        | `[var first, .., var last]`      | La secuencia tiene la estructura dada       |
| Lista `..`   | `[..]`, `[x, ..]`                | Segmento que puede tener cualquier longitud |
| `var`        | `var x`                          | Siempre coincide; captura el valor en x     |
| Wildcard     | `_`                              | Siempre coincide; no captura                |
| Guardia      | `pattern when condicion`         | El patrón coincide Y la condición es true   |

## Resumen

```
¿Quiero elegir entre múltiples valores constantes?
    └─► Constant pattern:  valor switch { 1 => ..., 2 => ..., _ => ... }

¿Quiero ramificar según el tipo del objeto?
    └─► Type pattern:      obj switch { string s => ..., int n => ..., _ => ... }

¿Quiero verificar propiedades del objeto?
    └─► Property pattern:  obj switch { { Edad: > 18, Nombre: "Ana" } => ... }

¿Quiero comparar rangos numéricos?
    └─► Relational pattern: n switch { < 0 => ..., >= 0 and < 10 => ..., _ => ... }

¿Quiero combinar condiciones en el patrón?
    └─► Logical:  and, or, not

¿Quiero examinar varios valores a la vez?
    └─► Tuple pattern: (a, b) switch { (true, false) => ..., _ => ... }

¿Quiero desestructurar un record o clase?
    └─► Positional pattern: punto switch { (0, 0) => ..., (var x, var y) => ... }

¿Quiero analizar la estructura de una colección?
    └─► List pattern: lista switch { [] => ..., [var x] => ..., [var h, ..] => ... }

¿El patrón no es suficiente y necesito una condición extra?
    └─► Guardia when: { Prop: valor } when condicionExtra => resultado

¿Quiero capturar un valor para usarlo en la guardia o el resultado?
    └─► var pattern: { Prop: var v } when v > 0 => $"valor: {v}"
```
