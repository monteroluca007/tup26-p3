# Tipos compuestos y colecciones en C#

Hasta ahora trabajamos con tipos simples: `int`, `double`, `bool`, `string`. Cada variable guarda un único valor. Pero en casi cualquier programa real hace falta agrupar o coleccionar datos. Para eso existen los **tipos compuestos**.

En este documento recorremos las estructuras más importantes del ecosistema C#, ordenadas de menor a mayor complejidad:

| Tipo                 | ¿Para qué?                              | Tamaño    | Clave                        |
|----------------------|-----------------------------------------|-----------|------------------------------|
| Tupla `(T1, T2, ...)`| Agrupar valores temporalmente           | Fijo      | Liviana, sin clase           |
| Array `T[]`          | Secuencia homogénea de tamaño fijo      | Fijo      | Acceso O(1) por índice       |
| `List<T>`            | Secuencia dinámica                      | Dinámico  | Agrega y quita elementos     |
| `Dictionary<K,V>`    | Mapa clave → valor                      | Dinámico  | Búsqueda O(1) por clave      |
| `HashSet<T>`         | Colección sin duplicados                | Dinámico  | Pertenencia eficiente        |
| `record`             | Dato con identidad semántica            | Fijo      | Igualdad por valor           |
| Tipo anónimo         | Proyección temporal                     | Fijo      | Solo dentro del método       |

## Arrays

Un array guarda varios valores del **mismo tipo** en posiciones contiguas de memoria. El tamaño queda definido al crearlo y no cambia. Los índices comienzan en `0`.

### Creación

```cs
// Con valores — sintaxis tradicional
int[] numeros = new int[] { 10, 20, 30, 40 };

// Compacta
int[] pares = { 2, 4, 6, 8 };

// C# 12+: literal de colección (recomendado)
int[] notas = [7, 3, 9, 5, 8];

// Con inferencia de tipo
var dias = new[] { "lun", "mar", "mié", "jue", "vie" };

// Array vacío con tamaño — valores por defecto (0, null, false según el tipo)
int[] vacio = new int[5];
```

### Acceso a elementos

```cs
int[] numeros = [10, 20, 30, 40, 50];

int primero   = numeros[0];               // 10
int segundo   = numeros[1];               // 20
int ultimo    = numeros[numeros.Length - 1]; // 50
int penultimo = numeros[^2];              // 40 — índice desde el final
```

Intentar acceder a una posición inexistente produce un error en tiempo de ejecución. Si el array tiene 5 elementos, la última posición válida es `4`.

### Modificación de elementos

A diferencia de `string`, los arrays permiten modificar sus elementos directamente:

```cs
int[] numeros = { 10, 20, 30 };
numeros[1] = 99;
// { 10, 99, 30 }
```

### Slicing con rangos

La sintaxis `inicio..fin` extrae una porción del array. El índice `fin` es exclusivo; `^n` cuenta desde el final.

```cs
int[] numeros = [10, 20, 30, 40, 50, 60];

var primerosTres = numeros[0..3];   // [10, 20, 30]
var ultimosDos   = numeros[^2..];   // [50, 60]
var medio        = numeros[2..^2];  // [30, 40]
```

### Recorrido

```cs
int[] numeros = [10, 20, 30, 40];

// foreach — cuando no hace falta el índice
foreach (int numero in numeros)
    Console.WriteLine(numero);

// for — cuando hace falta trabajar con la posición
for (int i = 0; i < numeros.Length; i++)
    Console.WriteLine($"[{i}] {numeros[i]}");
```

### Propiedades y métodos de utilidad

`Length` es una propiedad de instancia. Muchas operaciones de ordenamiento, búsqueda y copia son métodos estáticos de la clase `Array`. Además, si está disponible LINQ, aparecen extensiones como `Contains`:

```cs
int[] notas = [7, 3, 9, 5, 8, 2];

int cantidad = notas.Length;                     // 6

Array.Sort(notas);                               // [2, 3, 5, 7, 8, 9]
Array.Reverse(notas);                            // [9, 8, 7, 5, 3, 2]

int posicion   = Array.IndexOf(notas, 7);        // índice de la primera ocurrencia
bool hayMayor  = Array.Exists(notas, n => n > 8); // true
bool contiene  = notas.Contains(5);              // true

var copia = (int[])notas.Clone();                // copia independiente
Array.Copy(notas, copia, 3);                     // copia los primeros 3 elementos
```

### Listado ampliado de propiedades, métodos y operaciones sobre arrays

Para que el panorama quede completo, conviene separar tres grupos distintos:

1. **Miembros propios del array**
2. **Métodos estáticos de `System.Array`**
3. **Métodos de extensión de LINQ**

> En los cuadros siguientes se listan los nombres más importantes **sin detallar todas las sobrecargas**. En .NET varios de estos métodos tienen múltiples variantes.

| **GetLowerBound**    | `matriz.GetLowerBound(0)`              | Límite inferior de una dimensión.                                  |
| **GetUpperBound**    | `matriz.GetUpperBound(0)`              | Límite superior de una dimensión.                                  |

#### 3. Extensiones LINQ que también se usan con arrays
Tambien existe muchos metodos de extensión de LINQ que se pueden usar con arrays, como `Where`, `Select`, `OrderBy`, `Count`, `First`, etc.

#### Por qué muchas operaciones son métodos estáticos

Un detalle que suele llamar la atención: se escribe `Array.Sort(numeros)` y no `numeros.Sort()`. La razón es que un array es un tipo especial del runtime con pocas propiedades propias. Las operaciones de ordenamiento, búsqueda o copia son algoritmos generales aplicables a cualquier array, por eso .NET las concentra en la clase `Array` como una caja de herramientas común. Esto contrasta con `List<T>`, diseñada desde el inicio como una estructura más rica que expone esas mismas operaciones como métodos de instancia.

### Arrays multidimensionales

```cs
// Rectangular — todas las filas con el mismo largo
int[,] matriz =
{
    { 1, 2, 3 },
    { 4, 5, 6 },
    { 7, 8, 9 }
};
Console.WriteLine(matriz[1, 2]); // 6

int [,,] cubo = // Array tridimensional 
[
    [
        [1, 2],
        [3, 4]
    ],
    [
        [5, 6],
        [7, 8]
    ]
];
Console.WriteLine(cubo[0, 1, 0]); // 3

(byte r, byte g, byte b)[,] imagen = // Array de tuplas
[
    [(255,   0,   0), (  0, 255,   0)],
    [(  0,   0, 255), (255, 255,   0)]
];
Console.WriteLine(imagen[0, 1]); // (0, 255, 0)



// Jagged (array de arrays) — cada fila puede tener distinto largo
int[][] triangulo = new int[3][];
triangulo[0] = [1];
triangulo[1] = [1, 2];
triangulo[2] = [1, 2, 3];
```

### Qué idea debería quedar

Un array es fundamental para entender cómo se organizan datos en memoria: acceso directo O(1) por posición, recorrido secuencial, tamaño fijo. Si la cantidad de elementos no se conoce de antemano o varía en ejecución, la alternativa es `List<T>`.

## Colecciones: `List<T>`

`List<T>` es la colección dinámica más usada de .NET. A diferencia del array, puede crecer o reducirse en tiempo de ejecución. La `T` es el **parámetro de tipo**: al escribir `List<string>` el compilador garantiza que solo entren `string`.

### Creación

```cs
// Vacía
var nombres = new List<string>();

// Con valores — sintaxis de colección C# 12+ (recomendado)
List<string> ciudades = ["Tucumán", "Córdoba", "Rosario", "Mendoza"];

// Desde array existente
int[] arr  = [1, 2, 3, 4];
var  lista = arr.ToList();

// Con capacidad inicial (optimización cuando se conoce el tamaño aproximado)
var grande = new List<int>(capacity: 10_000);
```

### Diferencias clave con los arrays

- El tamaño es dinámico: se pueden agregar y quitar elementos.
- Se usa `Count` en lugar de `Length` para obtener la cantidad de elementos.
- Expone métodos de instancia directamente: `Add`, `Remove`, `Sort`, etc.

### Acceso y modificación

```cs
List<string> items = ["Alpha", "Beta", "Gamma"];

// Leer
string primero = items[0];
int    cantidad = items.Count;
bool   existe   = items.Contains("Beta");
int    indice   = items.IndexOf("Beta");

// Modificar
items[1] = "BETA";

// Agregar
items.Add("Delta");
items.Insert(0, "Omega");            // inserta en posición 0
items.AddRange(["Epsilon", "Zeta"]); // agrega varios a la vez

// Eliminar
items.Remove("Gamma");               // primera aparición de "Gamma"
items.RemoveAt(0);                   // por posición
items.RemoveAll(x => x.Length > 4); // todos los que cumplan la condición
items.Clear();                       // vacía la lista
```

### Recorrido

```cs
List<int> numeros = [10, 25, 30];

foreach (int numero in numeros)
    Console.WriteLine(numero);

for (int i = 0; i < numeros.Count; i++)
    Console.WriteLine($"[{i}] {numeros[i]}");
```

### Operaciones de ordenamiento

```cs
List<int> numeros = [30, 10, 25];

numeros.Sort();    // [10, 25, 30]
numeros.Reverse(); // [30, 25, 10]
```

### LINQ — consultas sobre colecciones

`List<T>` implementa `IEnumerable<T>`, lo que habilita todo el ecosistema LINQ:

```cs
record Producto(string Nombre, decimal Precio, int Stock);

List<Producto> productos =
[
    new("Monitor 27\"",     450_000, 12),
    new("Teclado mecánico",  85_000, 40),
    new("Mouse inalámbrico", 35_000,  0),
    new("Webcam HD",         55_000,  8),
    new("Auriculares",       70_000,  0),
];

// Filtrar + ordenar + proyectar
var enStock = productos
    .Where(p => p.Stock > 0)
    .OrderBy(p => p.Precio)
    .Select(p => $"{p.Nombre}: ${p.Precio:N0}");

// Estadísticas
decimal total    = productos.Where(p => p.Stock > 0).Sum(p => p.Precio * p.Stock);
int     sinStock = productos.Count(p => p.Stock == 0);

// Agrupar
var grupos = productos
    .GroupBy(p => p.Stock > 0 ? "disponible" : "sin stock")
    .ToDictionary(g => g.Key, g => g.ToList());
```

> **Evaluación diferida:** las consultas LINQ no se ejecutan al definirlas; se ejecutan al iterar el resultado (`foreach`, `ToList()`, `Count()`, etc.). Esto se llama *lazy evaluation*.

## Tipos genéricos

Al trabajar con `List<int>` o `List<string>`, la `T` entre los ángulos representa el tipo del elemento. Los **tipos genéricos** permiten escribir código que funciona para distintos tipos sin duplicarlo.

El problema que resuelven se entiende bien con un ejemplo. Una función que calcule el promedio de números requeriría una copia para cada tipo:

```cs
int   Promedio(int[]   numeros) { ... }
float Promedio(float[] numeros) { ... }
long  Promedio(long[]  numeros) { ... }
```

Con un parámetro de tipo se escribe una sola versión que funciona para todos:

```cs
T Promedio<T>(T[] numeros) where T : INumber<T>
{
    T suma = T.Zero;
    foreach (T numero in numeros)
        suma += numero;
    return suma / T.CreateChecked(numeros.Length);
}
```

La restricción `where T : INumber<T>` le indica al compilador que `T` debe ser un tipo numérico, lo que habilita las operaciones aritméticas. `List<T>`, `Dictionary<TKey, TValue>` y `HashSet<T>` son todos tipos genéricos: la misma implementación sirve para enteros, cadenas, objetos propios o cualquier otro tipo.

## Tuplas

Una tupla agrupa varios valores, posiblemente de distintos tipos, sin necesidad de definir una clase.

### Creación

```cs
// Con nombres de campo — recomendado
var persona = (Nombre: "Ada", Edad: 20, Activa: true);
Console.WriteLine(persona.Nombre); // Ada
Console.WriteLine(persona.Edad);   // 20

// Inferencia de nombres desde variables (C# 10+)
var nombre = "Ana";
var edad   = 25;
var p = (nombre, edad);   // p.nombre, p.edad

// Sin nombres — acceso por posición (evitar)
var tupla = ("Ana", 30, true);
Console.WriteLine(tupla.Item1); // "Ana"
Console.WriteLine(tupla.Item2); // 30
```

> El tipo real es `System.ValueTuple<T1, T2, ...>`. Los nombres de campo son azúcar sintáctica del compilador y no existen en tiempo de ejecución.

### Cuándo sirven

Son útiles para devolver varios valores desde un método, agrupar datos relacionados de forma rápida o hacer prototipos intermedios, sin que valga la pena definir un tipo dedicado.

### Retorno múltiple de métodos

```cs
static (double Min, double Max, double Promedio) Estadisticas(int[] datos)
{
    return (datos.Min(), datos.Max(), datos.Average());
}

var (min, max, prom) = Estadisticas([3, 7, 1, 9, 2]);
Console.WriteLine($"Min: {min}, Max: {max}, Promedio: {prom:F2}");
```

### Deconstrucción

Se puede descomponer una tupla en variables individuales. El guion bajo `_` descarta un campo que no interesa:

```cs
var punto = (X: 10.5, Y: 3.2);

var (x, y)      = punto;    // ambos campos
var (_, soloY)  = punto;    // solo Y
```

### Pattern matching

```cs
static string Clasificar((int Edad, bool EsVip) cliente) => cliente switch
{
    (>= 18, true)  => "Adulto VIP",
    (>= 18, false) => "Adulto estándar",
    (< 18,  _)     => "Menor de edad"
};

Console.WriteLine(Clasificar((22, true)));  // Adulto VIP
Console.WriteLine(Clasificar((15, true)));  // Menor de edad
```

### Cuándo NO usar tuplas

| Situación                                  | Mejor alternativa |
|--------------------------------------------|-------------------|
| El dato pasa entre varias capas del sistema | `record`          |
| Tiene comportamiento propio (métodos)      | `class` o `record` |
| Se usa en colecciones públicas             | `record` o `class` |

## Tipos anónimos

Un tipo anónimo crea un objeto con propiedades sin declarar previamente una clase con nombre. El compilador genera internamente el tipo; el código fuente nunca ve su nombre, por eso la variable **debe** declararse con `var`.

### Creación

```cs
var alumno = new { Nombre = "Ana", Edad = 21 };

Console.WriteLine(alumno.Nombre); // Ana
Console.WriteLine(alumno.Edad);   // 21
Console.WriteLine(alumno);        // { Nombre = Ana, Edad = 21 }
```

### Inferencia de nombres de propiedad

Si el valor proviene de una variable existente, el nombre se infiere automáticamente:

```cs
string nombre = "Carlos";
int    edad   = 30;

var p = new { nombre, edad }; // p.nombre, p.edad
```

### Igualdad estructural

Dos instancias con las mismas propiedades (mismo nombre, tipo y orden) se comparan por valor con `Equals`. El operador `==` no está sobrecargado para tipos anónimos:

```cs
var a = new { Nombre = "Ana",  Edad = 25 };
var b = new { Nombre = "Ana",  Edad = 25 };
var c = new { Nombre = "Luis", Edad = 30 };

Console.WriteLine(a.Equals(b)); // True
Console.WriteLine(a.Equals(c)); // False
```

### Caso de uso principal — proyecciones LINQ

Los tipos anónimos están diseñados como destino de `Select` cuando se necesita un subconjunto de campos y no vale la pena crear un `record` dedicado:

```cs
record Empleado(string Nombre, string Departamento, decimal Salario, int Antiguedad);

List<Empleado> empleados =
[
    new("Ana",    "IT",       95_000, 5),
    new("Carlos", "IT",       87_000, 3),
    new("Laura",  "RRHH",     72_000, 8),
    new("Diego",  "RRHH",     68_000, 2),
    new("Sofía",  "Finanzas", 110_000, 10),
];

var resumen = empleados
    .Where(e => e.Salario > 70_000)
    .OrderByDescending(e => e.Salario)
    .Select(e => new
    {
        e.Nombre,
        e.Departamento,
        SalarioFormateado = $"${e.Salario:N0}",
        Seniority = e.Antiguedad >= 5 ? "Senior" : "Junior"
    });

foreach (var item in resumen)
    Console.WriteLine($"{item.Nombre,-10} | {item.Departamento,-10} | {item.SalarioFormateado,12} | {item.Seniority}");
```

### Limitaciones importantes

Las propiedades son de solo lectura una vez creado el objeto:

```cs
var punto = new { X = 1, Y = 2 };
punto.X = 5; // Error de compilación: la propiedad es de solo lectura
```

| Limitación                                      | Alternativa      |
|-------------------------------------------------|------------------|
| No se puede usar como parámetro de método tipado| `record` o clase |
| No se puede retornar desde un método tipado     | `record` o tupla |
| No se puede usar fuera del método donde se creó | `record`         |

### Tipos anónimos vs. tuplas vs. records

| Característica                  | Tipo anónimo      | Tupla            | `record`       |
|---------------------------------|-------------------|------------------|----------------|
| Nombres de field                | Sí                | Opcional         | Sí             |
| Igualdad por value              | `.Equals()`       | `==`             | `==`           |
| Usable fuera del método         | No                | Sí               | Sí             |
| Retornable desde método tipado  | No                | Sí               | Sí             |
| Herencia                        | No                | No               | Sí             |
| Expresión `with`                | No                | Parcial          | Sí             |
| Uso principal                   | Proyecciones LINQ | Retorno múltiple | DTOs / Dominio |

## Records

Un `record` es un tipo especial introducido en C# 9 diseñado para representar **datos con identidad semántica**. El compilador genera automáticamente: constructor, propiedades, `Equals`, `GetHashCode`, `ToString` y deconstrucción.

### Diferencia clave con `class`

```cs
// Con class — igualdad por referencia
class PersonaClass { public string Nombre; public int Edad; }

var c1 = new PersonaClass { Nombre = "Ana", Edad = 25 };
var c2 = new PersonaClass { Nombre = "Ana", Edad = 25 };
Console.WriteLine(c1 == c2); // False — dos objetos distintos en memoria

// Con record — igualdad por valor
record PersonaRecord(string Nombre, int Edad);

var r1 = new PersonaRecord("Ana", 25);
var r2 = new PersonaRecord("Ana", 25);
Console.WriteLine(r1 == r2); // True — mismo contenido = iguales
```

### Declaración

```cs
// Posicional — la forma más compacta (recomendada)
record Persona(string Nombre, int Edad);

// Con cuerpo — permite agregar miembros calculados
record Punto(double X, double Y)
{
    public double Distancia => Math.Sqrt(X * X + Y * Y);
    public Punto Trasladar(double dx, double dy) => this with { X = X + dx, Y = Y + dy };
}

// Record struct — vive en el stack (para datos pequeños y de alto rendimiento)
readonly record struct Color(byte R, byte G, byte B);
```

### Instanciación y `ToString`

```cs
var p = new Persona("Ana", 25);

Console.WriteLine(p.Nombre); // Ana
Console.WriteLine(p.Edad);   // 25
Console.WriteLine(p);        // Persona { Nombre = Ana, Edad = 25 }
```

### Expresión `with` — copiar con cambios

Como los records son inmutables por convención, para "modificar" uno se crea una copia con los campos que cambian:

```cs
var original   = new Persona("Ana", 25);
var cumpleaños = original with { Edad = 26 };

Console.WriteLine(original.Edad);   // 25 — no se modificó
Console.WriteLine(cumpleaños.Edad); // 26
```

Este patrón se llama **copy-on-write** y es fundamental en arquitecturas inmutables.

### Deconstrucción

```cs
var persona = new Persona("Ana", 25);
var (nombre, edad) = persona;

Console.WriteLine($"{nombre} tiene {edad} años");
```

### Herencia y pattern matching

Los records admiten herencia, lo que los hace muy expresivos combinados con switch expressions:

```cs
abstract record Figura(string Color);
record Circulo(string Color, double Radio)                  : Figura(Color);
record Rectangulo(string Color, double Ancho, double Alto)  : Figura(Color);
record Triangulo(string Color, double Base, double Altura)  : Figura(Color);

static double CalcularArea(Figura fig) => fig switch
{
    Circulo    { Radio: var r }              => Math.PI * r * r,
    Rectangulo { Ancho: var a, Alto: var h } => a * h,
    Triangulo  { Base: var b, Altura: var h} => b * h / 2,
    _                                        => throw new ArgumentException($"Figura desconocida: {fig}")
};
```

### Guía de uso

| Contexto                                           | Tipo recomendado        |
|----------------------------------------------------|-------------------------|
| Retorno temporal de un método                      | Tupla                   |
| DTO entre capas (API, servicio, repositorio)       | `record`                |
| Mensaje de dominio / evento                        | `record`                |
| Resultado con éxito/error                          | `record` con herencia   |
| Entidad con estado mutable                         | `class`                 |
| Dato pequeño y de alto rendimiento                 | `readonly record struct`|

## Diccionarios: `Dictionary<TKey, TValue>`

Un diccionario almacena pares **clave → valor** donde cada clave es única. Internamente usa una tabla hash que permite acceso en O(1): encontrar un valor por su clave no requiere recorrer la colección.

### Creación

```cs
// Sintaxis de inicializador con índice (recomendado)
var telefonos = new Dictionary<string, string>
{
    ["Ana"]    = "381-123-4567",
    ["Carlos"] = "381-987-6543",
    ["Laura"]  = "381-555-0000"
};

// Sintaxis con llaves
Dictionary<string, int> edades = new Dictionary<string, int>
{
    { "Ana",   30 },
    { "Juan",  25 },
    { "Pedro", 35 }
};

// Desde una lista con LINQ
var porNombre = productos.ToDictionary(p => p.Nombre);
var porClave  = productos.ToDictionary(p => p.Nombre, p => p.Precio);
```

### Acceso

```cs
// Acceso directo — lanza KeyNotFoundException si la clave no existe
string numero = telefonos["Ana"];

// TryGetValue — acceso seguro (recomendado)
if (telefonos.TryGetValue("Pedro", out var tel))
    Console.WriteLine($"Teléfono: {tel}");

// GetValueOrDefault — valor de fallback si la clave no existe
var tel2 = telefonos.GetValueOrDefault("Pedro", "Sin número");

bool existe = telefonos.ContainsKey("Ana");
```

### Modificación

```cs
telefonos.Add("Diego", "381-111-2222");      // lanza si la clave ya existe
telefonos["Ana"]  = "381-999-8888";          // agrega o actualiza
telefonos.TryAdd("Ana", "381-000-0000");     // no hace nada si ya existe (devuelve false)
telefonos.Remove("Carlos");
```

### Recorrido

```cs
// Deconstrucción del par clave/valor — más legible
foreach (var (nombre, telefono) in telefonos)
    Console.WriteLine($"{nombre}: {telefono}");

// Acceso explícito a Key y Value
foreach (var par in telefonos)
    Console.WriteLine($"{par.Key}: {par.Value}");

// Solo claves o solo valores
foreach (var clave in telefonos.Keys)   Console.WriteLine(clave);
foreach (var valor in telefonos.Values) Console.WriteLine(valor);

// LINQ sobre el diccionario
var ordenados = telefonos
    .OrderBy(kv => kv.Key)
    .Select(kv => $"{kv.Key} → {kv.Value}");
```

### Métodos útiles

| Método / propiedad              | Qué hace                                       |
|---------------------------------|------------------------------------------------|
| `Add(clave, valor)`             | Agrega. Lanza si la clave ya existe.           |
| `TryAdd(clave, valor)`          | Agrega solo si la clave no existe. Devuelve `bool`. |
| `Remove(clave)`                 | Elimina el par.                                |
| `ContainsKey(clave)`            | `true` si la clave existe.                     |
| `ContainsValue(valor)`          | `true` si el valor existe en algún par.        |
| `TryGetValue(clave, out valor)` | Acceso seguro: `true` si encontró la clave.    |
| `GetValueOrDefault(clave, def)` | Devuelve el valor o `def` si no existe la clave. |
| `Keys`                          | Colección de todas las claves.                 |
| `Values`                        | Colección de todos los valores.                |
| `Count`                         | Cantidad de pares.                             |
| `Clear()`                       | Elimina todos los pares.                       |

### Variantes útiles

| Tipo                    | Diferencia                                      |
|-------------------------|-------------------------------------------------|
| `Dictionary<K,V>`       | Estándar, sin garantía de orden de inserción.   |
| `SortedDictionary<K,V>` | Claves ordenadas — O(log n) en acceso.          |
| `ConcurrentDictionary<K,V>` | Seguro para acceso desde múltiples hilos. |
| `OrderedDictionary<K,V>` | Mantiene orden de inserción — .NET 9+.          |

### Ejemplo: contar palabras en un texto

```cs
string texto = "hola mundo hola universo hola";
string[] palabras = texto.Split(' ');

Dictionary<string, int> conteo = new Dictionary<string, int>();

foreach (string palabra in palabras)
{
    if (conteo.TryGetValue(palabra, out int cantidad))
        conteo[palabra] = cantidad + 1;
    else
        conteo[palabra] = 1;
}

foreach (var (palabra, cantidad) in conteo)
    Console.WriteLine($"{palabra}: {cantidad}");
```

## Conjuntos: `HashSet<T>`

Un conjunto almacena elementos únicos: no permite duplicados. Cuando se intenta agregar un elemento que ya existe, la operación no tiene efecto. Internamente también usa una tabla hash, por lo que la verificación de pertenencia (`Contains`) es O(1).

### Creación

```cs
HashSet<int> numeros = new HashSet<int>();
numeros.Add(10);
numeros.Add(20);
numeros.Add(10); // ignorado — ya existe
// contiene { 10, 20 }

// Desde una colección existente — elimina duplicados automáticamente
string[] palabras = { "hola", "mundo", "hola", "universo" };
HashSet<string> unicas = new HashSet<string>(palabras);
// { "hola", "mundo", "universo" }
```

### Verificación de pertenencia

```cs
HashSet<string> frutas = new HashSet<string> { "manzana", "banana", "naranja" };

if (frutas.Contains("banana"))
    Console.WriteLine("Está en el conjunto.");
```

### Operaciones de conjuntos

`HashSet<T>` soporta las operaciones matemáticas clásicas. Todas modifican el conjunto receptor en su lugar:

```cs
HashSet<int> conjuntoA = new HashSet<int> { 1, 2, 3, 4 };
HashSet<int> conjuntoB = new HashSet<int> { 3, 4, 5, 6 };

var union        = new HashSet<int>(conjuntoA);
union.UnionWith(conjuntoB);            // { 1, 2, 3, 4, 5, 6 }

var interseccion = new HashSet<int>(conjuntoA);
interseccion.IntersectWith(conjuntoB); // { 3, 4 }

var diferencia   = new HashSet<int>(conjuntoA);
diferencia.ExceptWith(conjuntoB);      // { 1, 2 }
```

### Cuándo conviene usar un `HashSet`

Cuando lo que importa es saber si un elemento pertenece o no a la colección, sin importar el orden ni la posición. La verificación con `Contains` en un `HashSet` es muy eficiente incluso con millones de elementos, a diferencia de recorrer un array o lista para buscar.

### Ejemplo: palabras únicas en un texto

```cs
string texto = "hola mundo hola universo";
HashSet<string> unicas = new HashSet<string>(texto.Split(' '));
Console.WriteLine($"Palabras únicas: {unicas.Count}"); // 3
```

## Resumen comparativo

### Tabla de características

| Estructura          | Tipo homogéneo | Tamaño fijo | Acceso por clave | Sin duplicados | Igualdad por valor |
|---------------------|:--------------:|:-----------:|:----------------:|:--------------:|:------------------:|
| `T[]` (array)       | ✓              | ✓           | —                | —              | —                  |
| `List<T>`           | ✓              | —           | —                | —              | —                  |
| `Dictionary<K,V>`   | —              | —           | ✓                | claves únicas  | —                  |
| `HashSet<T>`        | ✓              | —           | —                | ✓              | —                  |
| Tupla               | —              | ✓           | —                | —              | ✓                  |
| Tipo anónimo        | —              | ✓           | —                | —              | `.Equals()`        |
| `record`            | —              | ✓           | —                | —              | ✓ (`==`)           |

### Árbol de decisión

```
¿Qué necesito?
│
├─ Texto o construcción de texto en loop
│   └─► string / StringBuilder
│
├─ Agrupar valores temporalmente en un método (sin clase)
│   └─► Tupla
│
├─ Dato con identidad semántica, que viaja entre capas
│   └─► record
│
├─ Proyectar campos temporalmente dentro de una consulta LINQ
│   └─► Tipo anónimo
│
├─ Secuencia de elementos del mismo tipo
│   ├─ Tamaño conocido en compilación → Array T[]
│   └─ Tamaño dinámico              → List<T>
│
├─ Buscar por clave en O(1)
│   └─► Dictionary<TKey, TValue>
│
└─ Garantizar unicidad / operaciones de conjuntos
    └─► HashSet<T>
```
