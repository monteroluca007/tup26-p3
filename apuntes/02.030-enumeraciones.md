# Enumeraciones en C#

## Introducción

Un **enum** (enumeración) es un tipo que define un conjunto finito de constantes con nombre. Reemplaza números o strings mágicos por valores con significado explícito:

```csharp
// Sin enum — ¿qué significa el 2? ¿y el "A"?
if (orden.Estado == 2) { ... }
if (usuario.Rol == "A") { ... }

// Con enum — el código se explica a sí mismo
if (orden.Estado == EstadoOrden.Enviado) { ... }
if (usuario.Rol == RolUsuario.Administrador) { ... }
```

Los enums en C# son **tipos por valor** basados en un tipo entero subyacente (`int` por defecto). Cada constante nombrada es simplemente un entero con un nombre legible.

---

## 1. Declaración básica

```csharp
// Enum simple — valores asignados automáticamente: 0, 1, 2, 3...
public enum DiaSemana
{
    Lunes,      // 0
    Martes,     // 1
    Miercoles,  // 2
    Jueves,     // 3
    Viernes,    // 4
    Sabado,     // 5
    Domingo     // 6
}

// Enum con valores explícitos
public enum EstadoOrden
{
    Pendiente  = 0,
    Confirmado = 1,
    Enviado    = 2,
    Entregado  = 3,
    Cancelado  = 99   // valor especial — no necesitan ser consecutivos
}

// Enum con tipo subyacente distinto de int
public enum CodigoError : short
{
    Ninguno      = 0,
    Generico     = 1000,
    NoEncontrado = 1001,
    SinPermiso   = 1002,
    Timeout      = 1003,
}
```

### 1.1 Tipos subyacentes permitidos

| Tipo                      | Rango         | Cuándo usarlo                              |
|---------------------------|---------------|--------------------------------------------|
| `int` (default)           | −2.1B a 2.1B  | Caso general                               |
| `byte`                    | 0 a 255       | Pocos valores, serialización compacta      |
| `short`                   | −32K a 32K    | Valores medianos                           |
| `long`                    | ±9.2×10¹⁸     | Cuando se necesitan flags con muchos bits  |
| `uint`, `ushort`, `ulong` | sin signo     | Flags con muchos valores positivos         |

## 2. Uso básico

```csharp
// Asignación
DiaSemana hoy = DiaSemana.Miercoles;
var mañana    = DiaSemana.Jueves;    // var infiere DiaSemana

// Comparación
if (hoy == DiaSemana.Sabado || hoy == DiaSemana.Domingo)
    Console.WriteLine("Es fin de semana");

// Switch expression — la forma idiomática con enums
string tipo = hoy switch
{
    DiaSemana.Sabado or DiaSemana.Domingo => "Fin de semana",
    DiaSemana.Lunes                       => "Inicio de semana",
    DiaSemana.Viernes                     => "¡Viernes!",
    _                                     => "Día hábil"
};

// En colecciones
var laborales = new List<DiaSemana>
{
    DiaSemana.Lunes, DiaSemana.Martes, DiaSemana.Miercoles,
    DiaSemana.Jueves, DiaSemana.Viernes
};
```

## 3. Conversión entre enum y sus tipos subyacentes

```csharp
// Enum → int (cast explícito)
int valor = (int)EstadoOrden.Enviado;    // 2

// int → enum (cast explícito)
EstadoOrden estado = (EstadoOrden)2;     // EstadoOrden.Enviado

// ⚠ Cuidado: un valor sin nombre asignado no lanza excepción
EstadoOrden raro = (EstadoOrden)50;      // válido — aunque 50 no esté definido

// Enum → string
string nombre = DiaSemana.Lunes.ToString();     // "Lunes"
string nombre2 = nameof(DiaSemana.Lunes);       // "Lunes" — en tiempo de compilación

// string → enum — Parse (lanza FormatException si no existe)
DiaSemana dia = Enum.Parse<DiaSemana>("Martes");

// string → enum — TryParse (seguro, no lanza)
if (Enum.TryParse<DiaSemana>("Viernes", out DiaSemana resultado))
    Console.WriteLine($"Parseado: {resultado}");

// TryParse ignora mayúsculas si se pide
Enum.TryParse<DiaSemana>("lunes", ignoreCase: true, out DiaSemana d);
```

### 3.1 Verificar si un valor es válido

```csharp
// Enum.IsDefined — verifica que el valor corresponda a un miembro definido
bool valido1 = Enum.IsDefined<EstadoOrden>(EstadoOrden.Enviado);  // true
bool valido2 = Enum.IsDefined<EstadoOrden>((EstadoOrden)50);      // false

// Útil para validar datos externos (JSON, base de datos, API)
static EstadoOrden ValidarEstado(int valorExterno)
{
    if (!Enum.IsDefined<EstadoOrden>((EstadoOrden)valorExterno))
        throw new ArgumentOutOfRangeException(nameof(valorExterno),
            $"Estado inválido: {valorExterno}");
    return (EstadoOrden)valorExterno;
}
```

## 4. Reflexión sobre enums — `Enum` estático

La clase estática `System.Enum` provee herramientas para inspeccionar enums en tiempo de ejecución:

```csharp
// Obtener todos los valores

    Console.WriteLine($"{(int)dia}: {dia}");

int cantidad = Enum.GetValues<DiaSemana>().Length;   // 7

// LINQ sobre los valores del enum
var diasLaborales = Enum.GetValues<DiaSemana>()
    .Where(d => d != DiaSemana.Sabado && d != DiaSemana.Domingo)
    .ToList();
```

## 5. Atributos en enums — `[Description]` y display

Para mostrar textos más amigables que el nombre del miembro, se usan atributos:

```csharp
using System.ComponentModel;

public enum EstadoPedido
{
    [Description("Pendiente de pago")]
    PendientePago    = 0,

    [Description("Pago confirmado")]
    PagoConfirmado   = 1,

    [Description("En preparación")]
    EnPreparacion    = 2,

    [Description("Enviado al cliente")]
    Enviado          = 3,

    [Description("Entregado")]
    Entregado        = 4,

    [Description("Cancelado por cliente")]
    Cancelado        = 99,
}

// Método de extensión para obtener la descripción
public static class EnumExtensions
{
    public static string Descripcion(this Enum valor)
    {
        var tipo  = valor.GetType();
        var campo = tipo.GetField(valor.ToString());
        var attr  = campo?.GetCustomAttributes(typeof(DescriptionAttribute), false)
                         .OfType<DescriptionAttribute>()
                         .FirstOrDefault();
        return attr?.Description ?? valor.ToString();
    }
}

// Uso
Console.WriteLine(EstadoPedido.EnPreparacion.Descripcion());
// "En preparación"

// Generar un menú desde el enum
foreach (EstadoPedido estado in Enum.GetValues<EstadoPedido>())
    Console.WriteLine($"[{(int)estado}] {estado.Descripcion()}");
```

## 6. Flags — enums de bit a bit

Cuando un valor puede ser **combinación** de varios miembros, se usa el atributo `[Flags]`. Cada miembro debe ser potencia de 2 para que los bits no se superpongan:

```csharp
[Flags]
public enum Permiso
{
    Ninguno  = 0,
    Leer     = 1,       // 0001
    Escribir = 2,       // 0010
    Ejecutar = 4,       // 0100
    Eliminar = 8,       // 1000

    // Combinaciones predefinidas
    LeerEscribir = Leer | Escribir,             // 0011 = 3
    Completo     = Leer | Escribir | Ejecutar | Eliminar,  // 1111 = 15
}

// Crear combinaciones
Permiso admin    = Permiso.Completo;
Permiso invitado = Permiso.Leer;
Permiso editor   = Permiso.Leer | Permiso.Escribir;

// Verificar si un permiso está incluido
bool puedeEscribir = (editor & Permiso.Escribir) != 0;   // true
bool puedeEliminar = (editor & Permiso.Eliminar) != 0;   // false

// HasFlag — más legible para verificar
bool puedeEscribir2 = editor.HasFlag(Permiso.Escribir);  // true
bool puedeEliminar2 = editor.HasFlag(Permiso.Eliminar);  // false

// Agregar un permiso
editor |= Permiso.Ejecutar;

// Quitar un permiso
editor &= ~Permiso.Escribir;

// ToString con [Flags] muestra los nombres combinados
Console.WriteLine(editor);    // "Leer, Ejecutar"
Console.WriteLine(admin);     // "Leer, Escribir, Ejecutar, Eliminar"
```

### 6.1 Método de extensión para flags

```csharp
public static class PermisosExtensions
{
    public static bool Tiene(this Permiso permisos, Permiso verificar)
        => permisos.HasFlag(verificar);

    public static Permiso Agregar(this Permiso permisos, Permiso nuevo)
        => permisos | nuevo;

    public static Permiso Quitar(this Permiso permisos, Permiso quitar)
        => permisos & ~quitar;
}

// Uso fluido
var permisos = Permiso.Leer
    .Agregar(Permiso.Escribir)
    .Agregar(Permiso.Ejecutar)
    .Quitar(Permiso.Escribir);

Console.WriteLine(permisos.Tiene(Permiso.Leer));      // true
Console.WriteLine(permisos.Tiene(Permiso.Escribir));  // false
```

## 7. Pattern matching con enums

```csharp
public enum Prioridad { Baja, Media, Alta, Critica }

record Incidente(string Titulo, Prioridad Prioridad, bool EstaResuelto);

// Switch expression con enums y guardias
static TimeSpan TiempoRespuesta(Incidente inc) => inc switch
{
    { EstaResuelto: true }                           => TimeSpan.Zero,
    { Prioridad: Prioridad.Critica }                 => TimeSpan.FromMinutes(15),
    { Prioridad: Prioridad.Alta }                    => TimeSpan.FromHours(1),
    { Prioridad: Prioridad.Media }                   => TimeSpan.FromHours(4),
    { Prioridad: Prioridad.Baja }                    => TimeSpan.FromDays(1),
    _                                                => throw new ArgumentOutOfRangeException()
};

// Pattern con múltiples valores
static string ColorIndicador(Prioridad p) => p switch
{
    Prioridad.Critica or Prioridad.Alta => "rojo",
    Prioridad.Media                     => "amarillo",
    Prioridad.Baja                      => "verde",
    _                                   => "gris"
};
```

### 8.4 `FileMode` y `FileAccess` — archivos

```csharp
using System.IO;

// FileMode — cómo abrir o crear el archivo
// FileAccess — qué se puede hacer con él
// FileShare — qué pueden hacer otros procesos simultáneamente

using var stream = new FileStream(
    "datos.bin",
    FileMode.OpenOrCreate,   // crea si no existe, abre si existe
    FileAccess.ReadWrite,    // leer y escribir
    FileShare.Read           // otros procesos pueden leer simultáneamente
);

// FileMode como flags de comportamiento
var modo = FileMode.Create;   // siempre crea nuevo (sobreescribe si existe)

// Ejemplo práctico — modo de apertura según la intención
static FileStream AbrirArchivo(string ruta, bool soloLectura)
{
    var modo   = soloLectura ? FileMode.Open   : FileMode.OpenOrCreate;
    var acceso = soloLectura ? FileAccess.Read  : FileAccess.ReadWrite;
    var share  = soloLectura ? FileShare.Read   : FileShare.None;
    return new FileStream(ruta, modo, acceso, share);
}
```

## 9. Enums como estado de máquinas

Los enums son ideales para modelar el estado en una **máquina de estados**:

```csharp
public enum EstadoConexion
{
    Desconectado,
    Conectando,
    Conectado,
    Reconectando,
    Error
}

public class ClienteWebSocket
{
    private EstadoConexion _estado = EstadoConexion.Desconectado;

    public EstadoConexion Estado => _estado;

    public async Task ConectarAsync(string url)
    {
        if (_estado != EstadoConexion.Desconectado)
            throw new InvalidOperationException($"No se puede conectar desde el estado {_estado}.");

        _estado = EstadoConexion.Conectando;
        try
        {
            await Task.Delay(100);   // simula handshake
            _estado = EstadoConexion.Conectado;
            Console.WriteLine("Conectado");
        }
        catch
        {
            _estado = EstadoConexion.Error;
            throw;
        }
    }

    public void Desconectar()
    {
        if (_estado != EstadoConexion.Conectado)
            throw new InvalidOperationException($"No está conectado (estado: {_estado}).");

        _estado = EstadoConexion.Desconectado;
    }

    // Las transiciones válidas definidas explícitamente
    private static readonly Dictionary<EstadoConexion, EstadoConexion[]> _transiciones = new()
    {
        [EstadoConexion.Desconectado]  = [EstadoConexion.Conectando],
        [EstadoConexion.Conectando]    = [EstadoConexion.Conectado, EstadoConexion.Error],
        [EstadoConexion.Conectado]     = [EstadoConexion.Desconectado, EstadoConexion.Reconectando],
        [EstadoConexion.Reconectando]  = [EstadoConexion.Conectado, EstadoConexion.Error],
        [EstadoConexion.Error]         = [EstadoConexion.Desconectado],
    };

    private void Transicionar(EstadoConexion destino)
    {
        if (!_transiciones[_estado].Contains(destino))
            throw new InvalidOperationException(
                $"Transición inválida: {_estado} → {destino}");
        _estado = destino;
    }
}
```

## 10. Serialización de enums

### 10.1 Con `System.Text.Json`

```csharp
using System.Text.Json;
using System.Text.Json.Serialization;

public enum TipoDocumento { DNI, Pasaporte, LC, LE }

public record Persona(string Nombre, TipoDocumento Documento);

// Por defecto — serializa como número
var p1      = new Persona("Ana", TipoDocumento.Pasaporte);
var json1   = JsonSerializer.Serialize(p1);
// {"Nombre":"Ana","Documento":1}

// Como string — más legible
var opciones = new JsonSerializerOptions
{
    Converters = { new JsonStringEnumConverter() }
};
var json2 = JsonSerializer.Serialize(p1, opciones);
// {"Nombre":"Ana","Documento":"Pasaporte"}

// Deserializar
var persona = JsonSerializer.Deserialize<Persona>(json2, opciones);

// Atributo en el tipo — aplica siempre
[JsonConverter(typeof(JsonStringEnumConverter))]
public enum EstadoPago { Pendiente, Aprobado, Rechazado }
```

## 11. Buenas prácticas

### 11.1 Definir un valor 0 significativo (o `Ninguno`)

```csharp
// MAL — 0 es el default de int pero no tiene nombre
public enum Color { Rojo = 1, Verde = 2, Azul = 3 }
Color c = default;   // 0 — no es un valor definido en el enum

// BIEN — 0 siempre tiene un nombre
public enum Color { Ninguno = 0, Rojo = 1, Verde = 2, Azul = 3 }
Color c = default;   // Color.Ninguno — valor definido y explícito
```

### 11.2 Siempre validar valores externos

```csharp
// Datos de red, base de datos o entrada del usuario
static T ParsearEnum<T>(string texto) where T : struct, Enum
{
    if (Enum.TryParse<T>(texto, ignoreCase: true, out T resultado) &&
        Enum.IsDefined(resultado))
        return resultado;

    throw new ArgumentException($"Valor inválido para {typeof(T).Name}: '{texto}'");
}
```

### 11.3 Preferir switch exhaustivo

```csharp
// Marcar el switch para que el compilador advierta si falta un caso
static string Etiqueta(DiaSemana dia) => dia switch
{
    DiaSemana.Lunes    => "L",
    DiaSemana.Martes   => "Ma",
    DiaSemana.Miercoles => "Mi",
    DiaSemana.Jueves   => "J",
    DiaSemana.Viernes  => "V",
    DiaSemana.Sabado   => "S",
    DiaSemana.Domingo  => "D",

};
```

### 11.5 Documentar los valores del enum

/// <summary>Estado del ciclo de vida de una orden de compra.</summary>

## Resumen

```
¿Necesito representar un conjunto finito de opciones con nombre?
    └─► enum

¿El valor puede ser combinación de varias opciones?
    └─► [Flags] enum con potencias de 2

¿Quiero mostrar un texto amigable por cada valor?
    └─► [Description] + método de extensión Descripcion()

¿Necesito iterar todos los valores posibles?
    └─► Enum.GetValues<T>()

¿Quiero parsear un string a enum de forma segura?
    └─► Enum.TryParse<T>() + Enum.IsDefined()

¿Necesito serializar el enum como texto en JSON?
    └─► JsonStringEnumConverter o [JsonStringEnumMemberName]

¿Quiero modelar estados y transiciones válidas?
    └─► enum + diccionario de transiciones + switch exhaustivo

Ejemplos clave de .NET:
    DayOfWeek         → días de la semana
    ConsoleColor      → 16 colores de consola
    HttpStatusCode    → códigos HTTP (200, 404, 500...)
    FileMode          → comportamiento al abrir archivos
    StringComparison  → cómo comparar strings
    RegexOptions      → [Flags] para opciones de regex
    TaskStatus        → estado de una Task asíncrona
    JsonValueKind     → tipos de nodos JSON
    SeekOrigin        → posicionamiento en streams
```
