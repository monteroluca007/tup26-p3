# Cómo entregar el trabajo práctico ?

---
1. **Se debe clonar el repositorio actual del curso.**
> [!TIP] Github Desktop: 
**File > Clone repository > URL > Repository URL: `alejandrodibattista/tup26-p3` > Clone**
---
2. **Se debe crear una rama nueva con el formato `tp-01/nombre-apellido`.**

    _(ejemplo: `tp-01/sofia-sanchez`)._
> [!TIP] Github Desktop: 
**Branch > New Branch > Name: `tp-01/nombre-apellido` > Create Branch**
---
3. **Se debe implementar el programa modificando el archivo `sortx.cs`, que se encuentra en la carpeta del alumno dentro de `practicos`.**

    (ejemplo: `practicos/63313 - Sanchez, Sofia/TP1/sortx.cs`).
> [!TIP] Github Desktop: 
**Repository > Open in Visual Studio Code**

> [!TIP] VS Code:
**Archivo > Abrir Carpeta > `practicos/63313 - Sanchez, Sofia/TP1/`**
---
4. **Se debe hacer commit de los cambios parciales, con al menos 5 commits y mensajes descriptivos, y luego hacer push a la rama creada.**

    (nota: solo se debe hacer commit de los archivos modificados dentro de la carpeta del trabajo práctico del alumno).
> [!TIP] Github Desktop: 
**Changes > Summary: `Mensaje descriptivo` > Commit to `tp-01/nombre-apellido`**
---
5. **Publicar la rama.**
> [!TIP] Github Desktop: 
**Repository > Push**
---
6. **Se debe abrir un Pull Request a la rama `main` con el formato `TP 01 - Legajo - Nombre Apellido`**
_(ejemplo: `TP 01 - 62000 - Sofia Sanchez`)._
> [!TIP] Github Desktop: 
**Branch > Create Pull Request > Title: `TP 01 - Legajo - Nombre Apellido` > Create Pull Request**
---
7. **Regresar a la rama `main` y actualizar el repositorio local.**
> [!TIP] Github Desktop: 
**Current Branch > main, Fetch origin**

---

> [!CAUTION] ATENCION: 
No se debe modificar ningún archivo fuera de la carpeta del alumno. Solo se debe modificar el archivo dentro de la carpeta del trabajo práctico del alumno. Cualquier cambio fuera de esa carpeta puede afectar a otros compañeros y no será considerado para la corrección del trabajo práctico.
